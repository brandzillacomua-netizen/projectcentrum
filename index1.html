FortnetAPI documentation¶

Введение¶

Задачи:

  • обмен данными по персоналу с существующей кадровой системой

  • предоставление данных по учету рабочего времени

Механизм обмена

Передача данных через HTTP-соединение. API Rest-style: идентификация объекта по URI, операция (прочитать/обновить) через указание метода (GET, PUT). Данные в формате xml или json.

Например:

http://fortnetservername/departments/ GET - получить список отделов

http://fortnetservername/departments/3/ GET получить отдел 3, PUT - обновить свойства отдела

http://fortnetservername/persons/7/ GET получить описание сотрудника 7, PUT - обновить свойства сотрудника 7

Например:

GET http://localhost/departments/2/ (текущие данные)

Response body:

{"department":
  {
   "Name": "Технический отдел",
   "GID": "2",
   "ParentDepGID": "4"
  }
}

PUT http://localhost/departments/2/ (изменяем название)

Request body:

{"department":
  {
   "Name": "Технический отдел 1",
   "GID": "2",
   "ParentDepGID": "4"
  }
}

GET http://localhost/departments/2/ (данные изменены)

Response body:

{"department":
  {
   "Name": "Технический отдел 1",
   "GID": "2",
   "ParentDepGID": "4"
  }
}

Инсталляция сервиса¶

Сервис интеграции может работать в режиме командной строки (приложение) или как сервис Windows.

Запуск в режиме командной строки

FortnetImportSvc.exe /console  (либо -c)

Инсталляция / деинсталляция сервиса Windows

FortnetImportSvc.exe /install [/uninstall]

Дополнительно - команды управления сервисом (запуск/останов/состояние)

FortnetImportSvc.exe /start [/stop  /status]

Сервис использует файл fortnet.ini для полученеия параметров (путь к базе данных). Кроме того, в разделе HTTPService могут быть настройки

[HTTPService]
Port=8090

Порт если не указан, то считается стандартный (80).

Проверка:

  1. в окне броузера набрать http://localhost/departments/ (предполагая, что fortnet запущена на локальной машине на порту 80). Результат сохранить и просмотреть в текстовом редакторе. Чтобы результат отображался (а не предлагалось сохранить), необходимо в настройках броузера указать обработку файлов json (для Firefox проще поставить расширение JSONView).

  2. Утилита RESTTester (скачать: http://www.ed.com.ua/files/RESTTester.zip) предназначена для проверки общения с сервером Fortnet.

Идентификаторы для связи¶

Для обмена данных необходимо, чтобы 2 системы могли однозначно идентифицировать объект каким-то образом. В БД Fortnet сотрудник Петров имеет идентификатор, допустим, 20, а в кадровой системе, например, 25. Каким образом кадровая система может указать исправить опечатку в его отчестве?

Для этого в таблицах отделы и персонал Fortnet добавлено поле “внешний идентификатор” (строка длиной 32 байт максимум). Значение поля должно быть уникальным. В качестве значений могут использоваться:

  • идентификаторы БД из кадровой системы

  • естественные ключи (табельный номер, ИНН и т.д.)

  • сгенерированные GUID (что-то похожее на ‘D87B944644BE47C18A397AB42C16E840’)

Эти идентификаторы устанавливаются внешней (кадровой) системой.

Локальные и глобальные идентификаторы¶

Локальные идентификаторы (LID, Local Identifiers) - идентификаторы, утанавливаемые системой Fortnet

Глобальные идентификаторы (GID, Global Identifiers) - идентификаторы, устанавливаемые внешней кадровой системой.

Для доступа к объектам по локальным идентификаторам необходимо указать в URI приставку local, например:

http://fortnetservername/local/departments/
http://fortnetservername/local/departments/id/  (id - локальный!)
http://fortnetservername/local/persons/id/   (id - локальный!)

Экспозиция локальных идентификаторов сделана по 2 причинам:

  1. Интеграция с системами, которые не ведут свою базу данных сотрудников

  2. Первоначальная “привязка” данных (возникла необходимость интеграции с кадровой системой, когда данные частично были уже внесены в Fortnet). См. подробнее ниже.

Первоначальная “привязка” данных¶

Общий алгоритм:

  1. Получить локальный идентификатор

  2. Записать объект (обновить) по глобальному идентификатору, передав в качестве дополнительного параметра локальный идентификатор (LID).

Например:

  1. http://fortnetservername/local/departments/ - получаем список отделов с локальными идентификаторами.

[
  {"name":"Интегратор-Плюс", "LID":"1", "ParentDepLID":"1"},
  {"name":"Коммерческий отдел", "LID":"3", "ParentDepLID":"1"},
...
]

  1. Предположим, в кадровой системе есть аналогичные объекты со своими идентификаторами

Интегратор-Плюс - id=20317

Коммерческий отдел - id=20319

Необходимо “увязать” их, чтобы не создавать дубликаты.

Направляем запрос

  1. http://fortnetservername/departments/20317/

Метод PUT

Header:

Content-type: application/json

Тело запроса:

{"name":"Интегратор-Плюс", "LID":"1", "ParentDepGID":"20317"}

  1. http://fortnetservername/departments/20319/

Метод PUT

Header:

Content-type: application/json

Тело запроса:

{"name":"Коммерческий отдел", "LID":"3", "ParentDepGID":"20317"}

Поддерживаемые URI¶

Ниже “id” в URI заменяется на идентификатор объекта

  • http://fortnetservername/version/ - Запрос версии API

  • http://fortnetservername/departments/ - Список отделов (структура организации)

  • http://fortnetservername/departments/id/ - Описание отдела

  • http://fortnetservername/departments/id/persons/ - Список сотрудников отдела

  • http://fortnetservername/persons/id/ - Описание сотрудника

  • http://fortnetservername/persons/id/photo/ - Фотография сотрудника

  • http://fortnetservername/persons/id/worktime/yyyy/mm/dd/ - Данные по рабочему времени сотрудника за дату

  • http://fortnetservername/accesslevels/ - Список уровней доступа

  • http://fortnetservername/hardwares/ - Список аппаратуры

  • http://fortnetservername/hardwares/id/stat/ - Cостояние устройства

  • http://fortnetservername/hardwares/stat/ - Список состояния аппаратуры

  • http://fortnetservername/online/ - Получение текущих событий (состояния)

  • http://fortnetservername/cards/id/ - Описание карточки доступа

  • http://fortnetservername/cards/ - Список карточек доступа

  • http://fortnetservername/wtschedules/ - Список рабочих графиков

  • http://fortnetservername/wtexceptkinds/ - Список типов исключений рабочих графиков

  • http://fortnetservername/persons/id/wtexcept/ - Регистрация исключения из рабочего графика сотрудника

  • http://fortnetservername/persons/id/wtschedule/ - Рабочий график сотрудника

  • http://fortnetservername/events/ - Запрос архива событий

Фильтр списка объектов¶

При запросе списка объектов (GET) в теле досумента может быть указан фильтр (в настоящее время поддерживается фильтр по дате последних изменений).

Формат (json):

{
    "Filter":
      {
        "dtChangedAfter": string
      }
}

Например:

{
    "Filter":
      {
        "dtChangedAfter": "19.03.2014 13:02:46"
      }
}

Поддерживаемые методы: GET, PUT

Формат (json):

Cостояние устройства¶

URI: http://fortnetservername/hardwares/id/stat/

id - локальный идентификатор

Методы:

  • GET прочитать состояние устройства

  • PUT послать команду управления

Метод PUT имеет смысл только для устройств, режимом которых можно управлять (Reader, Relay, Alarmzone, Alarmgroup).

Формат (json):

{
    "HardwareStat": {
        param: string,
        ...
    }
}

Параметры, описывающие состояние, зависят от типа устройства (kind)

Параметры для типов:

  • Module

  • Controller

  • ModuleDoor

  • Door

  • Reader

  • Alarmzone

  • Relay

  • Alarmgroup stat

  • Lift

  • Tambour

Пример:

GET http://fortnetservername/hardwares/11/stat/

Result body:

{
    "HardwareStat": {
        "AC": "Absent",
        "DC": "Low",
        "Tamper": "Alarm",
        "Connect": "Online",
        "Door": "Closed"
    }
}

Получение текущих событий (состояния)¶

URI: http://fortnetservername/online/

Методы: только GET

При подключении выдается Список состояния аппаратуры, а затем события

Описание события (пример):

{
    "Event": {
        "DateTime": "27.09.2011 19:04:43",
        "card": 1500289,
        "message": {
            "LID": "1",
            "kind": "PASSAGE_PRESENT",
            "Name": "Доступ разрешен"
        },
        "hardware": {
            "LID": "10",
            "Name": "Вход в техотдел",
            "hardwareStat": {
                "ReaderMode": "Duty"
            }
        },
        "person": {
            "LID": "4",
            "FullName": "Клименчук Сергей "
        }
    }
}

Параметры:

DateTime - дата и время card - код карточки доступа (если есть) message - тип события (см [[eventkinds|Типы событий]])

Описывется:

  • LID - идентификатор типа события (по базе данных)

  • kind - строковый идентификатор

  • Name - текстовое представление типа

  • comment - расшифровка (например, причина не предоставления доступа)

hardware - источник события (аппаратура) Формат см.описании аппаратуры person - сотрудник (если есть)

Описывется:

  • LID - идентификатор по БД

  • FullName - ФИО сотрудника

Описание карточки доступа¶

URI: http://fortnetservername/cards/id/

Поддерживаемые методы: GET (чтение), PUT (изменение)

{
    "Card": {
        "LID": string,
        "Active": string,
        "AntiPassBack": string,
        "Lost": string,
        "AccesslevelLID": number,
        "AlarmRightLID": number,
        "FloorRightLID": number,
        "CurrentAccessZoneLID": number,
        "OwnerGID": string,
        "OwnerLID": string,
        "Number": string,
        "StartDate": string,
        "EndDate": string,
        "Pincode": string,
    }
}

Параметры

  • LID - идентификатор (код карточки)

  • Active - активна

  • AntiPassBack- контроль повторного входа

  • Lost- утеряна

  • OwnerGID/OwnerLID- идентификатор (локальный идентификатор) владельца

  • AccesslevelLID- идентификатор уровня доступа

  • AlarmRightLID - идентификатор прав охранной сигнализации

  • FloorRightLID - идентификатор уровня доступа лифтов

  • CurrentAccessZoneLID - очистка статуса КПП карточки

  • Pincode - пинкод

  • Number - номер карточки (присваиваемый, для печати на бейджах)

  • StartDate - начало срока действия

  • EndDate - конец срока действия

        "StartDate": "20.03.2013",

Значение:

  • Alarm - тревога

  • Norm - норма

Значения:

  • Unknown - неизвестно

  • Inactive - не опрашивается;

  • Online - связь есть

  • Offline - связь отсутсвует
